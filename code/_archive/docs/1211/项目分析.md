  关键动画设计：

  1. 首页进场动画（GSAP Timeline）

  - Logo 从上方划入 + 星星闪烁（1s）
  - 标题文字渐入 + 轻微浮动（1.2s，stagger）
  - 按钮缩放进场 + 发光脉动（1.5s）
  - 背景粒子淡入（2s）

  2. Three.js 背景层

  - 粒子系统：3000-5000个粒子，模拟星空漂浮
  - 云雾效果：使用 Shader Material 做流动渐变
  - 交互响应：鼠标/触摸移动时，粒子产生微弱排斥效果

  3. 卡片网格动画（导航页）

  - 10个卡片 stagger 交错进场（每个间隔0.1s）
  - Hover 时：卡片上浮 + 边框发光 + 轻微倾斜
  - 点击时：放大并过渡到详情页

  4. 页面转场

  - 使用 Shared Element Transition
  - 卡片从网格位置平滑放大到全屏
  - 背景模糊度从0到10px过渡

  ---
  💡 性能优化建议

  针对触摸屏 Win10 机器：

  1. Three.js 优化
    - 粒子数量控制在5000以内
    - 使用 InstancedMesh 批量渲染
    - 帧率锁定60fps
  2. 图片优化
    - 插画用 WebP 格式（比PNG小30-50%）
    - 背景用渐进式加载
    - 首屏资源预加载
  3. GSAP 优化
    - 使用 will-change CSS属性
    - 避免同时运行超过10个动画
    - 用 transform 替代 left/top
  4. Electron 优化
    - 启用硬件加速
    - 禁用开发者工具
    - 打包时压缩资源

  🎯 核心亮点建议

  提升格调的关键点：

  1. 细腻的微交互
    - 按钮点击水波纹扩散
    - 搜索框聚焦时发光效果
    - 卡片悬停时周围粒子聚集
  2. 音效设计（可选）
    - 页面切换轻柔音效
    - 按钮点击反馈音
    - 背景环境音（图书馆需静音设计）
  3. 过渡动画
    - 使用 easings.net 的自然曲线
    - 推荐：Power3.easeOut 或 Expo.easeOut
  4. 触摸优化
    - 按钮区域放大到 60x60px 以上
    - 滑动手势支持
    - 长按返回主页

  ---
  ⚠️ 需要注意的问题

  1. 性能瓶颈：Three.js + 大量模糊效果可能吃性能
    - 解决：提供"性能模式"降低粒子数
  2. 触摸精度：1920x1080 分辨率下，UI元素不能太小
    - 建议：最小点击区域 48x48px
  3. 长时间运行：图书馆设备可能连续运行8小时+
    - 解决：添加内存释放、闲置返回首页（3分钟无操作）
